1 · System Architecture
Layered view of the platform. Two client types reach a single Next.js application (server-side rendered), which is organized as a monolith of interdependent domain modules over one PostgreSQL database and object storage. External services are called directly — no integration middleware.

2 · Data Flow — Quote-to-Cash & Web Store Orders
The core document flow. Orders originate either manually (Quote → Sales Order) or from the Web Store (auto-created SO with WEB- prefix within 60 seconds). From confirmation onward, both follow the same path to collected payment. Status changes sync back to the storefront.

3 · Data Flow — Content Library AI Pipeline
Every upload is analyzed by the Claude Vision API to generate tags and a rich description, which is embedded and stored in pgvector so assets are findable by natural-language search. If the AI call fails, the asset still uploads and is flagged for manual tagging.
