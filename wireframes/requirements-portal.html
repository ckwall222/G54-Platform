Project Overview

MakeReady by G54

● In Progress Great Mountain West · g54.com View live wireframes ↗

MakeReady is a purpose-built Print Management Information System (Print MIS) and ERP for Great Mountain West, a commercial print and graphics production company. It replaces three disconnected systems with a single integrated platform.

Replaced SystemFunctionMakeReady Equivalent
SAP Business OneERP — accounting, inventory, salesNative Finance, Sales, Inventory modules
Zoey B2B (greatmountainwest.zoey.com)B2B eCommerce storefrontNative Web Store at store.g54.com
Vision33 SaltboxiPaaS connector between SAP and ZoeyEliminated — direct integrations only

Sales

Sales — Quote to Cash

✓ Complete KLKim Lund View wireframe ↗

The Sales module manages G54's full quote-to-cash document flow. Every sales order — whether entered manually by a rep or received from the Web Store — follows the same path through to collected payment.

Document flow: Quote → Sales Order → Production Job (auto) → Delivery → AR Invoice → Incoming Payment

User Stories

US-SALES-01
Create a Quote
Complete
As a Sales Rep or Sales Manager
I want to create a price quote for a customer
So that I can present a formal price before they commit to an order
Acceptance Criteria
  • Given I create a quote, then I must select a Business Partner; the system applies their Account Group pricing to line items automatically
  • Given a discount exceeds 10%, then a Sales Manager approval flag is set before the quote can be sent
  • Given I save a quote, then it receives a system-generated number with prefix QUO-
  • Status flow: Draft → Sent → Accepted → Converted to SO | Declined | Expired
US-SALES-04
Web Store Order Becomes a Sales Order
Complete
As a Sales Manager
I want Web Store orders to appear as Sales Orders automatically
So that no manual re-entry is required for online orders
Acceptance Criteria
  • Given a customer places an order on store.g54.com, within 60 seconds a Sales Order appears in MakeReady with prefix WEB-
  • The SO contains: all line items, quantities, BP link, shipping address, and Web Store order reference
  • Given the order total is ≥ $5,000, then the SO is created in Pending Approval status and does not enter production until approved
US-SALES-08
Field Sales Order Submission (Mobile)
Complete
As a Sales Rep in the field
I want to create a Sales Order and attach client artwork from my tablet
So that the order is in the system before I leave the client's location
Acceptance Criteria
  • The sales order form is usable on a tablet (768px width) without horizontal scrolling
  • Given I tap "Attach Artwork," when I select a file from my device, then it uploads and attaches to the SO
  • Given I submit the SO, then I receive a confirmation screen with the new SO number

Every customer in MakeReady is a Business Partner (BP), mirroring the SAP Business One entity model. The Account Group assigned to a BP controls their Web Store pricing tier and catalog access automatically.

US-CRM-01

Complete

So that I can associate sales orders, invoices, and Web Store access with that customer

  • Duplicate company names show a warning but are allowed with confirmation

Account Groups
Complete
As a Sales Manager or Admin
I want to assign a Business Partner to an Account Group
So that their pricing tier and Web Store catalog access are automatically controlled
Acceptance Criteria
  • Given I change a BP's Account Group, then their Web Store pricing updates on next login
  • Account Group names and pricing rules: [TBD — to be defined by Kim Lund and Chase de Jong]
US-CRM-05
Activity Log
Complete
As a Sales Rep
I want to log notes, calls, emails, and visits against a customer
So that anyone on the team can see the full relationship history
Acceptance Criteria
  • Activity types: Note, Call, Email, Visit, Other
  • Activities are immutable once saved — no edit or delete
  • All team members' activities on a BP are visible to all reps (not just the author)

Jobs & Production

✓ Complete TJTyson Johnson View wireframe ↗

Every confirmed Sales Order automatically generates a production Job. Jobs move through a defined status pipeline from receipt to shipping.

Job status pipeline: New → Prepress → Printing → Finishing → Ready to Ship → Shipped

US-JOB-01
Auto-Create Job from Sales Order
Complete
As a Production member
I want a job to be created automatically when a Sales Order is confirmed
So that I don't have to re-enter details already in the SO
Acceptance Criteria
  • Within 30 seconds of SO confirmation, a Job with prefix JOB- appears in the production queue
  • Job contains: linked SO number, customer name, line items, quantities, and due date
  • Initial status is New
US-JOB-04
Attach Artwork to a Job
Complete
As an Art Department member
I want to attach approved artwork files to a job
So that Production has the correct files without searching for them
Acceptance Criteria
  • Artwork can be selected from the Content Library or uploaded directly (new uploads are added to the Library simultaneously)
  • Job shows a thumbnail preview and filename for each attached asset
  • Clicking the thumbnail opens the full-resolution file or a viewer

Art Group

Content Library — Digital Asset Management

✓ Complete CoCody de Jong J?Jon [TBD] View wireframe ↗

The Content Library gives the Art Department a single place to store, organize, and retrieve every piece of artwork and graphic. The defining capability is AI-powered natural language search — staff can describe what they're looking for in plain English and receive ranked, relevant results.

Example query: "find all graphics that feature a moose and look like they could fit into a national parks theme" — the AI interprets this into visual tags and searches across auto-generated image descriptions, returning results ranked by relevance.

US-LIB-02
AI Auto-Tagging on Upload
Complete
As an Art Department member
I want uploaded assets to be automatically tagged with descriptive labels
So that assets are searchable immediately without manual tagging effort
Acceptance Criteria
  • AI analysis completes within 60 seconds of upload; at least 3 auto-generated tags are applied
  • Tags capture: primary subjects, setting/location, color palette, style/aesthetic, and mood
  • Auto-tags are visually distinguished from manual tags in the UI
  • If AI analysis fails, the asset still uploads successfully and is flagged for manual tagging
US-LIB-03
Natural Language Search
Complete
As any MakeReady user with Content Library access
I want to search for assets using plain English descriptions
So that I can find relevant artwork without knowing filenames or tags
Acceptance Criteria
  • Natural language results return within 10 seconds for a library of up to 10,000 assets
  • Results are ranked by visual relevance (High / Medium / Low with percentage score)
  • A summary of how the AI interpreted the query is shown (e.g., "Searched for: wildlife featuring moose · national parks aesthetic · rustic color palette")
US-LIB-08
Client Assignment & Access Control
Complete
As an Art Department member
I want to assign assets to a specific client
So that client-owned artwork is organized by account and access is controlled
Acceptance Criteria
  • Sales Reps can only see assets assigned to their own clients, plus G54-owned stock
  • "Client-owned (restricted)" assets are not visible to users outside the client's sales rep and Art Department
